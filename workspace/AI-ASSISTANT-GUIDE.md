# 🎯 三省六部系统 - AI 助手集成指南

## 架构说明

```
用户
  ↕️ 自然语言对话
AI 助手 (OpenClaw Agent)
  ↕️ 调用执行引擎
三省六部系统
  ├─ 中书省 (决策)
  ├─ 门下省 (审核)
  ├─ 尚书省 (协调)
  └─ 六部 (执行)
```

## 使用方式

### 方式 1: 直接向 AI 助手下达任务

用户只需要说：
- "帮我创建一个 HTTP 服务器"
- "编写一个 REST API"
- "生成测试用例"
- "审查代码安全性"

AI 助手会：
1. 理解任务需求
2. 调用三省六部系统
3. 分解和执行任务
4. 返回结果给用户

### 方式 2: 批量任务管理

用户可以说：
- "帮我处理以下 3 个任务：..."
- "创建一个项目，包括服务器、测试和文档"

AI 助手会：
1. 分解成多个子任务
2. 按优先级排序
3. 依次执行
4. 汇总结果

## 任务类型

| 类型 | 说明 | 示例 |
|------|------|------|
| code | 代码开发 | "创建 HTTP 服务器" |
| test | 测试编写 | "编写单元测试" |
| security | 安全审查 | "检查代码漏洞" |
| config | 配置管理 | "设置环境变量" |
| docs | 文档编写 | "生成 API 文档" |
| deploy | 部署任务 | "部署到生产环境" |
| general | 通用任务 | "分析这个文件" |

## 优先级

- **urgent**: 紧急任务（立即执行，预算无上限）
- **high**: 高优先级（优先处理）
- **medium**: 中等优先级（默认）
- **low**: 低优先级（闲时处理）

## 成本控制

默认预算：$5/任务
- 可以指定预算："用 $2 创建一个服务器"
- 超预算会自动停止并询问
- 实时成本监控
